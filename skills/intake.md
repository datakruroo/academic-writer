## วิธีสัมภาษณ์
- ถามทีละ 1-2 คำถาม ไม่ถามทีเดียวทั้งหมด
- ถ้าคำตอบไม่ชัดเจน ให้ถามต่อหรือขอตัวอย่าง
- ใช้ภาษาที่เป็นธรรมชาติ

## ข้อมูลที่ต้องรวบรวม

### 1. Meta
{meta_fields}

### 2. Core Argument (สำคัญที่สุด)
- thesis: ข้อสรุปหรือคำถามหลักของงาน (1-2 ประโยคชัดเจน)
- key_claims: ข้ออ้างหลัก 3 ข้อที่สนับสนุน thesis

### 3. Structure
- default คือ: {default_structure}
- ถามว่าต้องการใช้ default หรือปรับแก้

### 4. Material
- ไฟล์ PDF อ้างอิงที่มีอยู่แล้ว (ถ้ายังไม่มีให้ข้าม)
- path ของ dataset ถ้ามี
- path ของ R/Python script ถ้ามี

### 5. Constraints
- ประเด็นที่ไม่ต้องการพูดถึง
- ประเด็นที่ต้องการเน้น
- คำแนะนำพิเศษด้านสไตล์

## เมื่อรวบรวมข้อมูลครบ
สรุปให้นักวิจัยตรวจสอบก่อน เมื่อได้รับการยืนยันให้เรียก write_file โดยใช้ path "intent.md" และเนื้อหาตาม template นี้ กรอกข้อมูลจริงทุกฟิลด์:

{intent_template}

หมายเหตุ structure_block:
- ถ้าใช้ default ให้ใส่: `# ใช้ default structure`
- ถ้าปรับแก้ให้ใส่รายการ sections จริง
